Comprehensions are useful as ways to simplify code. However, we have to be careful about writing WORN (Write Once, Read Never) code, as the comprehension syntax is unique to Python and so non-Python speakers will not understand it.

So be judicious in your use of comprehensions, and don’t use a single-line comprehension just to avoid writing a for loop across two lines.

For instance, consider how much more language-specific knowledge it takes to understand:

numList = [1,2,3,4,5]
[i ** 2 for i in numList]

vs.

numList = [1,2,3,4,5]
newList = numList
for i in range(len(numList)):
  numList[i] = numList[i] ** 2

The second is wordier, and that helps non-native python speakers, as the for loop construction is universal in all languages. Someone familiar with programming in general – but not python in specific– would be able to trivially understand the for loop code. Whereas the comprehension would be, well, incomprehensible.
