Thereâ€™s a concept called a deep copy and another one called a shallow copy. A deep copy is when everything in the object is copied over to the new object. The shallow copy is when only the reference to the object is copied. Letâ€™s look at how this works:

Note after the assignment how changing a doesnâ€™t affect b. Theyâ€™re totally separate objects.

Shallow copy#

Note how the two lists are not at all dependent!

# Set the myList
myList = ['1', '2', '3', '4']

# Create a new variable, `myNewList`, and assign `myList` to it.

# Print the old list

# Check that these two variables, `myList` and `myNewList` are indeed exactly the same!

Oh, okay, thatâ€™s a bit of a kick in the pants. So sometimes you think youâ€™ve made a deep copy, but thatâ€™s not always the case. So when objects are mutable, and you change just a single index in the object, you do not create a new object.

Honestly, itâ€™s a bit hard to explain, right now we just need you to understand that it happens.
